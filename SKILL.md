# Leon PPT 生成 Skill v1.0

基于 PptxGenJS，直接生成完整 `.pptx` 文件。支持多种版式、思维模型自动识别、AI品牌Logo嵌入。

---

## 简化两阶段工作流

```
Phase 1：内容分析 + 大纲确认
  1.1 → 分析内容结构，识别思维模型信号（PDCA/SWOT/黄金圈等）
  1.2 → 输出大纲，每页标注推荐版式
  1.3 → 用户确认/调整

Phase 2：内容脚本 + 生成交付
  2.1 → 输出逐页内容脚本
  2.2 → 用户确认 → 生成文件 → 视觉QA → 交付
```

---

## Phase 1：内容分析 + 大纲确认

### 1.1 内容特征识别

| 内容特征 | 匹配版式 | 信号关键词 |
|---------|---------|-----------|
| 单一结论 + 3个分论点 | 金字塔版式 | "结论先行"、"三大支柱" |
| 4格循环结构 | PDCA版式 | 计划/执行/检查/改进 |
| 2×2矩阵对比 | SWOT版式 | 优势/劣势/机会/威胁 |
| 三层嵌套逻辑 | 黄金圈版式 | WHY/HOW/WHAT |
| 6要素分布 | 5W1H版式 | Who/When/What/Where/Why/How |
| 4步因果链 | SCQA版式 | 场景→冲突→问题→解决 |
| 2-3个数据指标 | 数据卡片 | 百分比、增长率 |
| 时间线/里程碑 | 时间轴 | Q1→Q2→Q3→Q4 |
| 左右对比 | 对比栏 | 方案A vs 方案B |

### 1.2 大纲输出格式

```
──────────────────────────────────────────────────
  PPT大纲 · [主题] · 共 X 页
  版式分析：[PDCA × 1页 | 数据卡片 × 2页 | 要点列表 × N页]
──────────────────────────────────────────────────
第 01 页  [封面页]              主标题 / 副标题 / 日期
第 02 页  [目录页]              4个章节导览
第 03 页  [章节分隔 01]         章节名 / 副标题
第 04 页  [SWOT矩阵]            页面标题  理由：明确出现优势/劣势/机会/威胁四要素
第 05 页  [数据卡片]            页面标题  理由：包含3个百分比指标
...
第 XX 页  [结尾页]              致谢
──────────────────────────────────────────────────
```

### 1.3 用户确认

使用 `AskUserQuestion` 提问：
- 选「没问题」→ 进入 Phase 2
- 选「调整版式」→ 用户指定页码和新版式

## Phase 2：内容脚本 + 生成交付

### 2.1 内容脚本格式

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
【第 X 页】版式：[PDCA循环]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
标题：2025年度项目复盘

模型结构：
  P（计划）：年初制定的三大目标
  D（执行）：实际落地情况，完成率80%
  C（检查）：发现效率瓶颈在交付环节
  A（改进）：2026年引入自动化流程

排版指令：4格循环色块（顺时针）
图片：无
```

### 2.2 内容原则

- 每页 ≤5 个要点，单条 ≤30 字
- 内容不超过版面 70%，留白 ≥ 30%
- 相邻 3 页不得使用同一版式
- 每页至少包含一种视觉元素（图标、数字、色块）

### 2.3 生成文件

**执行流程：**
```
Read `layout-presets.md` → 获取版式代码模板
Read `pptx-builder.md`  → 获取构建流程规范
编写 build_pptx.js → node 执行 → 视觉QA → 交付
```

## 支持版式（22种）

**基础版式：** 封面页、目录页、章节分隔页、要点列表、数据卡片、左右对比、横向流程、图文并排、时间轴、结尾页

**数据展示：** 数据看板、Bento Grid、悬浮统计、对比栏

**思维模型：** 金字塔/MECE、PDCA循环、SWOT矩阵、黄金圈、5W1H、SCQA、IPD五看

## 常见情况处理

| 情况 | 处理方式 |
|------|---------|
| 用户直接提供完整内容 | 直接进入内容分析，输出大纲 |
| 用户明确指定模型 | 直接采用指定模型版式 |
| 用户说「直接生成」 | 快速模式：合并输出，末尾确认 |
| 用户只要内容脚本不要文件 | 完成脚本后停止 |
| 用户要求修改某页 | 定位代码，修改参数，重新生成 |
| 内容含 AI 品牌关键词 | 触发Logo嵌入，查映射表 |
